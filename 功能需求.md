# 功能需求

## 账单字段

每条账单建议保存这些信息：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| id | 是 | 每条记录的唯一编号 |
| type | 是 | 支出或收入，第一版可以只做支出 |
| amount | 是 | 金额 |
| category | 是 | 吃饭、娱乐、出行、日常生活等 |
| paymentMethod | 是 | 微信、支付宝、现金、银行卡等 |
| member | 是 | 这笔账属于哪个家庭成员 |
| note | 否 | 备注，比如“音乐软件续费” |
| rawText | 否 | 用户原始语音转文字内容 |
| date | 是 | 记账时间 |

## 首页

首页需要让用户一眼看到：

- 今日支出
- 本月支出
- 本月收入
- 最大支出类别
- 每个家庭成员本月花了多少钱
- 最近账单列表
- 一个明显的语音记账按钮

## 记账流程

1. 用户点击麦克风按钮。
2. App 提示用户开始说话。
3. 用户说一句自然语言。
4. App 转成文字。
5. App 自动解析账单。
6. App 展示确认页面。
7. 用户确认或修改金额、分类、支付方式、成员。
8. 保存账单。

## 家庭成员

第一版成员功能建议这样做：

- 默认有“我”和“家人”两个成员。
- 用户可以自己添加成员名字，比如“爸爸”“妈妈”“孩子”。
- 每笔账都必须归属到一个成员。
- 语音里如果出现成员名字，优先自动识别。
- 如果语音里没有成员名字，默认记到当前选中的成员。
- 删除成员时，不删除历史账单。

## 信息不完整时的处理

如果没有识别出金额：

- 不直接保存。
- 提示：“这条账单缺少金额，请补充。”

如果没有识别出支付方式：

- 默认填“其他”。
- 用户可以修改。

如果没有识别出类别：

- 默认填“日常生活”。
- 用户可以修改。
