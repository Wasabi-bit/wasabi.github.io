# 苹果手机语音记账软件

这是一个面向个人使用的语音记账项目。目标是让用户用自然语言说一句话，软件自动识别金额、用途、支付方式和类别，并保存成一条账单记录。

## 核心场景

用户说：

> 我用微信花了20元吃饭

软件自动记录：

| 字段 | 内容 |
| --- | --- |
| 金额 | 20 |
| 收支类型 | 支出 |
| 类别 | 吃饭 |
| 支付方式 | 微信 |
| 备注 | 吃饭 |

用户说：

> 我用支付宝续费了音乐软件

软件自动记录：

| 字段 | 内容 |
| --- | --- |
| 金额 | 需要追问或手动补充 |
| 收支类型 | 支出 |
| 类别 | 娱乐 |
| 支付方式 | 支付宝 |
| 备注 | 音乐软件续费 |

## 默认分类

- 吃饭：早餐、午餐、晚餐、外卖、咖啡、奶茶、零食、聚餐
- 娱乐：音乐会员、视频会员、游戏、电影、演出、书籍、玩具
- 出行：地铁、公交、打车、高铁、飞机、加油、停车
- 日常生活：超市、洗漱用品、衣服、房租、水电、手机费、快递、维修

## 默认支付方式

- 微信
- 支付宝
- 现金
- 银行卡
- 信用卡
- 其他

## 第一版 MVP 功能

第一版不要做得太复杂，先实现最重要的闭环：

1. 点击一个麦克风按钮开始说话。
2. 识别语音并转成文字。
3. 从文字里提取金额、类别、支付方式和备注。
4. 如果信息不完整，比如没有金额，提醒用户补充。
5. 用户确认后保存账单。
6. 首页展示今日支出、本月支出和最近账单。
7. 可以按类别查看统计。
8. 可以设置家庭成员，并记录每笔账属于谁。
9. 可以按成员查看本月支出。

## 推荐技术路线

### 路线 A：真正的 iPhone App

适合目标：以后想长期使用、上架 App Store、体验像正式软件。

建议技术：

- SwiftUI：做 iPhone 界面
- Apple Speech Framework：语音转文字
- 本地数据库 SwiftData 或 Core Data：保存账单
- 可选 AI 解析：把自然语言转换成结构化账单

优点：

- iPhone 体验最好
- 可以做本地通知、快捷入口、小组件
- 更适合长期维护

缺点：

- 需要 Mac 电脑和 Xcode
- 如果上架 App Store，需要苹果开发者账号
- 对小白来说学习成本较高

### 路线 B：先做网页原型

适合目标：先验证想法，不急着上架。

建议技术：

- HTML / CSS / JavaScript
- 浏览器语音输入或手动输入
- 本地存储账单数据

优点：

- 开发快
- 可以先把流程跑通
- 成本低

缺点：

- iPhone 上的语音能力和原生 App 比较有限
- 体验不如真正 App
- 后续如果要上架，还是要重做一部分

### 路线 C：无代码过渡版

适合目标：你想马上开始用，不想先学代码。

可以用：

- iPhone 快捷指令
- 备忘录 / Numbers / Notion / 飞书多维表格
- 语音输入加自动分类规则

优点：

- 最简单
- 很快能用
- 适合测试自己的真实记账习惯

缺点：

- 不像一个完整 App
- 自动分类和统计能力有限

## 难度评估

如果做一个能用的第一版，难度是中等。

对你这样的零基础用户来说，难点不在“记账”本身，而在三个地方：

1. iPhone App 开发环境：需要 Mac、Xcode、苹果开发者体系。
2. 语音识别：要处理权限、识别失败、用户说得不完整等情况。
3. 自然语言解析：同一句话可能有很多说法，比如“微信二十买午饭”“刚刚支付宝打车35”“奶茶用了18”。

大致难度：

| 功能 | 难度 |
| --- | --- |
| 手动记账 | 低 |
| 语音转文字 | 中 |
| 自动分类 | 中 |
| 统计图表 | 中 |
| iCloud 同步 | 中高 |
| 上架 App Store | 中高 |
| 多人共享账本 | 高 |

## 建议的开发顺序

1. 先确定账单字段和分类。
2. 做一个能手动添加账单的页面。
3. 加语音输入。
4. 加自动解析。
5. 加确认页面，避免识别错了直接保存。
6. 加首页统计。
7. 再考虑云同步、预算、导出 Excel、App Store 上架。

## 我们下一步可以做什么

建议下一步先做一个「网页交互原型」，让你在电脑或手机浏览器里看到软件长什么样、流程怎么走。确认体验满意后，再决定是否升级成真正的 iPhone App。

## 放到 iPhone 上使用

推荐路线：发布成一个私人使用的网页地址，然后用 iPhone Safari 添加到主屏幕。

### 为什么不直接传文件到手机

直接把 `index.html` 发到手机也许能打开，但体验不稳定：

- 不能像 App 一样从主屏幕打开。
- 离线缓存和图标配置不一定生效。
- 语音权限更容易受浏览器限制。

### 推荐方案：GitHub Pages

适合你现在这个项目，因为它是纯静态网页，不需要服务器。

大致步骤：

1. 注册或登录 GitHub。
2. 新建一个仓库，比如 `family-book`。
3. 上传当前文件夹里的这些文件：`index.html`、`styles.css`、`app.js`、`manifest.json`、`sw.js`、`app-icon.svg`、`apple-touch-icon.png`。
4. 在仓库设置里打开 Pages。
5. 发布后会得到一个网址。
6. 用 iPhone 的 Safari 打开这个网址。
7. 点分享按钮，选择“添加到主屏幕”。

注意：账单数据会保存在你手机浏览器本地。不要清除 Safari 网站数据，否则账单可能丢失。建议定期使用右上角导出功能备份 CSV。
