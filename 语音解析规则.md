# 语音解析规则

语音解析的目标是把一句自然语言转换成结构化账单。

## 输出格式

```json
{
  "type": "expense",
  "amount": 20,
  "category": "吃饭",
  "paymentMethod": "微信",
  "member": "我",
  "note": "吃饭",
  "needsReview": false,
  "missingFields": []
}
```

## 解析原则

1. 金额是最重要字段，没有金额时必须让用户确认或补充。
2. 支付方式优先从“微信”“支付宝”“现金”“银行卡”“信用卡”中识别。
3. 类别根据关键词判断。
4. 成员优先从家庭成员名字中识别。
5. 如果没有识别到成员，默认归属当前选中的成员。
6. 备注尽量保留用户说的消费对象。
7. 不确定时不要自作主张，标记 `needsReview`。

## 需要追问的情况

没有金额：

> 你这笔账缺少金额，请问花了多少钱？

没有类别：

> 我不确定这笔属于哪个类别，请选择一个分类。

没有支付方式：

> 我没有识别到支付方式，要记为“其他”吗？
